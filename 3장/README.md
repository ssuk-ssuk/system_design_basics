# 3장: 시스템 설계 면접 공략법

이 챕터는 정답이 없는 시스템 설계 면접을 어떻게 건설적으로 이끌어나갈 것인지에 대한 실무적인 가이드라인을 담고 있습니다. 면접자의 기술적 역량뿐만 아니라 협업 능력과 문제 해결 과정을 효과적으로 보여주는 흐름을 학습합니다.

---

## 🧠 면접 임하기: 핵심 마인드셋

- **정답은 없다:** 시스템 설계에는 고정된 결말이나 정답이 없음을 인지해야 합니다.

- **다양한 역량 표출:** 단순 설계 능력을 넘어 협력 적합성, 압박 상황 대처 능력, 모호한 문제 해결 능력을 보여주는 과정입니다.

- **효율적 시간 배분:** 제한된 시간 내에 핵심을 전달하는 것이 매우 중요합니다.

---

## 🗺️ 4단계 설계 프로세스

1. 문제 이해 및 설계 범위 확정

- **성급한 답변 금지:** 요구사항을 완전히 이해하지 않고 답을 내놓는 것은 부정적인 신호입니다.

- **올바른 질문하기:** 속도를 늦추고 질문을 통해 요구사항과 가정들을 분명히 하십시오.

2. 개략적인 설계안 제시 및 동의 구하기

- **청사진 제시:** 클라이언트, API, 웹 서버, 데이터 저장소, 캐시, CDN 등 핵심 컴포넌트를 포함하는 다이어그램을 그립니다.

- **협력적 태도:** 면접관을 팀원이라 생각하고 의견을 구하며 진행합니다.

3. 상세 설계

- **우선순위 설정:** 면접관과 상의하여 시스템의 성능, 병목 구간, 자원 요구량 등 집중할 주제를 좁힙니다.

- **핵심에 집중:** 불필요한 세부 사항에 시간을 낭비하지 말고 역량을 집중적으로 보여줍니다.

4. 마무리

- **비판적 사고:** 개선 가능한 지점을 언급하십시오. "완벽하다"는 태도는 비판적 사고 부족으로 보일 수 있습니다.

- **추가 제안:** 오류 대응, 미래의 규모 확장성, 미처 다루지 못한 세부 개선안을 요약하며 마무리합니다.

---

## 🛠️ 부록: 피드 서비스 설계 예시

해당 프로세스를 적용한 실제 사례로 **뉴스 피드 시스템**의 상세 설계를 다룹니다.

- **주요 구성 요소:** 포스팅 저장/전송 서비스, 알림 서비스, 뉴스 피드 캐시 등.

- **상세 기술 스택:** 처리율 제한 장치, 메시지 큐를 활용한 비동기 작업 서버, 친구 관계를 위한 그래프 데이터베이스 활용.
